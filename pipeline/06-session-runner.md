# スキル: セッションランナー

## トリガー: `/session`

## 概要

ガイド付きデイリー学習セッションを実行する。
ユーザーの学習時間に応じてセッション構成を動的に調整。

## ⚠️ 重要: ターン分割

セッションは対話形式で進行する。1ターンで全部やらない。

```
ターン1: ウォームアップ + 新教材の前半
ターン2: ユーザー回答 → フィードバック + 復習問題
ターン3: ユーザー回答 → フィードバック + ラップアップ
```

## 処理フロー

### Step 1: セッション準備
1. `data/profile.json` → 学習時間を確認
2. `data/progress.json` → 復習キュー + 現在の学習位置を確認
3. 今日のプラン（`data/sessions/plan_*.md`）があればそれに従う

### Step 2: セッション構成決定

#### 15分セッション
```
WARM-UP（2分）: 前回の要点を1つ思い出す
RETRIEVAL（8分）: 復習2-3問
WRAP-UP（5分）: 今日の学び記録
```

#### 30分セッション（推奨）
```
WARM-UP（3分）: 前回サマリーを記憶から書く
NEW MATERIAL（10分）: 新トピック教材（HOOK+ORIGIN+CORE）
RETRIEVAL（12分）: 復習3-4問（インターリービング）
WRAP-UP（5分）: 知識ダンプ + 自信度記録
```

#### 60分セッション
```
WARM-UP（5分）: 前回サマリー記憶 + 自信度評価
NEW MATERIAL（15分）: 新トピック教材
RETRIEVAL SPRINT（15分）: 復習5-6問（インターリービング）
DEEP THINKING（15分）: ソクラテス対話 or ケーススタディ
WRAP-UP（10分）: 知識ダンプ + ナレッジウェブ接続 + 自信度再評価
```

### Step 3: 各フェーズの実行

#### WARM-UP
- 「前回のセッションで学んだことを、見返さずに書いてください」
- ユーザー回答後、実際の内容と比較してギャップを指摘
- **この時点でメタ認知チェック**: 「今日の復習対象について、どの程度覚えていると思いますか？（1-5）」

#### NEW MATERIAL
- 未学習トピックの教材を表示（`/learn` の短縮版: HOOK+CORE のみ）
- 途中で精緻化質問を挟む: 「ここまでの内容から、なぜこれが重要だと思いますか？」
- ユーザーの回答を待ってから次に進む

#### RETRIEVAL SPRINT
- `04-learning-engine.md` の出題形式を使用
- 必ず異なる分野からインターリービング
- 各問題後に即座にフィードバック（遅延フィードバックはquizモードで使用）

#### DEEP THINKING（60分セッションのみ）
- ソクラテス式対話（3-4往復の短縮版）
- または「2つのトピックを接続せよ」の統合チャレンジ

#### WRAP-UP
- 「今日学んだことを30秒で書き出してください（知識ダンプ）」
- ストリーク更新
- 明日の予告（復習キューの次回対象を案内）

### Step 4: セッションログ保存
```json
// data/sessions/session_[日付].json
{
  "date": "2026-03-04",
  "duration_minutes": 30,
  "new_topics": ["T031"],
  "reviewed_topics": ["T011", "T008"],
  "scores": { "T011": 4, "T008": 3 },
  "metacognition": { "predicted": 4, "actual": 3.5, "gap": 0.5 },
  "streak_updated": true
}
```
