## `/plan` — 週間学習プラン生成

### 処理フロー

1. `data/progress.json` と `data/profile.json` を読む
2. 以下を考慮してプランを生成:
   - 1日の学習時間（profile.daily_minutes）
   - 復習キューの溜まり具合
   - 前提トピックの習得状態
   - 分野バランス（偏りがあれば是正提案）
3. `data/sessions/plan_[日付].md` に保存

### プラン設計原則

- **新規:復習 = 3:7**（復習を優先、これが間隔反復の効果を最大化する）
- **インターリービング**: 毎日異なる分野のトピックを混ぜる
- **望ましい困難**: 簡単すぎる日を作らない
- **休息日**: 週1日はオフ（過学習防止）

### 表示フォーマット

```
📅 週間学習プラン（3/4 - 3/10）

月: 🆕 T031 LLMアーキテクチャ(前半) + 🔄 T011復習 + 🔄 T008復習 [30分]
火: 🔄 T031復習 + 🔄 T023復習 + 🧠 T011ソクラテス [30分]
水: 🆕 T034 プロンプトエンジニアリング(前半) + 🔄 T031復習 [30分]
木: 🔄 T034復習 + 🔄 T011復習 + 📝 T008クイズ [30分]
金: 🆕 T034(後半) + 🔄 T023復習 + 🎓 T011ティーチ [30分]
土: 📊 週間レビュー + 統合チャレンジ [60分]
日: 休息日

→ `/session` でその日のセッションを開始
```

## モチベーション管理（自己決定理論ベース）

### 3つの基本的欲求への対応

| 欲求 | 設計 |
|------|------|
| 自律性 | 学習順序の選択権、スキップ可能、ペース調整 |
| 有能感 | レベルアップ通知、マイルストーン達成、具体的スコア |
| 関係性 | AI師匠との対話的学習、学びの共有促進 |

### ストリーク管理

- `profile.json` の `streak` を毎セッション更新
- last_date と今日の差が1日 → current++
- last_date と今日の差が2日以上 → current = 1
- マイルストーン: 3日/7日/14日/30日/90日で特別メッセージ

### マイルストーン通知（該当時のみ表示）

```
🎉 マイルストーン達成！
- 「初めての分野横断統合」: 2分野以上のトピックを接続できた
- 「10トピック学習」: 学習済みトピックが10を超えた
- 「過信度改善」: calibration_gap が ±0.5 以内に安定
- 「7日ストリーク」: 1週間連続で学習を継続
```
