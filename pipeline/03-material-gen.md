# スキル: 教材生成

## トリガー: `/learn [トピックID]`

## ⚠️ 重要: 3ターン分割設計

教材は1ターンで全部生成しない。以下の3ターンに分割する。

### ターン1: `/learn T0XX` → HOOK + ORIGIN + CORE
### ターン2: `/continue` → EVIDENCE + BUSINESS + AI LENS
### ターン3: `/continue` → MENTAL MODEL + CHALLENGE + FURTHER

各ターンの末尾で `data/materials/T0XX.md` に追記保存し、
「`/continue` で次のセクションを生成します」と案内する。

## 処理フロー

### Step 1: 前提チェック
- `data/progress.json` を読み、前提トピックの習得状態を確認
- 未習得の前提がある場合 → 警告（強制ではない）

### Step 2: Web検索（1-2回のみ）
- トピックに関する最新の重要論文・事例を1-2回検索
- 検索クエリ例: `"psychological safety" team performance meta-analysis`
- 検索結果は教材に反映するが、逐一引用はしない

### Step 3: 教材生成（当該ターン分のセクションのみ）

### Step 4: ファイル保存
- `data/materials/T0XX.md` に保存（追記モード）
- `data/_pending.json` に次ターン情報を保存:
  ```json
  { "command": "learn", "topic": "T0XX", "phase": 2, "next_sections": ["EVIDENCE","BUSINESS","AI_LENS"] }
  ```

### Step 5: 進捗更新
- `data/progress.json` のトピックを `level: 1 (EXPOSED)` に更新

## 教材テンプレート「知的冒険」

### セクション1: 🎯 HOOK（導入フック）
- 衝撃的事実、逆説的問い、または「常識の破壊」から始める
- 例: 「Googleが200チームを調査した結果、最強チームの共通点はスキルでも経験でもなかった」
- 最大3-4文。読者の好奇心を一気に掴む

### セクション2: 📖 ORIGIN STORY（起源と発見の物語）
- この概念がどう発見/発展したかのストーリー
- 研究者の人間ドラマ、失敗、ブレークスルーの瞬間
- なぜこの概念が重要になったのかの文脈
- 5-8文程度

### セクション3: 🧠 CORE THEORY（核心理論）
以下を構造的に記述:
- **定義**: 1-2文の明確な定義
- **メカニズム**: なぜ・どのように機能するのか
- **エビデンス強度**: メタ分析の効果量、サンプルサイズ等
- **境界条件**: どんな場面では機能しないか
- **主要な批判**: 学術的な反論、限界

### セクション4: 🔬 EVIDENCE SHOWCASE（エビデンス展覧会）
- 重要な研究を3-5件紹介
- 各研究: 著者(年) / 方法 / 発見 / インパクト / 意外な点
- メタ分析があれば優先的に紹介

### セクション5: ⚔️ BUSINESS BATTLEFIELD（ビジネス戦場）
- 成功事例1件 + 失敗事例1件（実在企業推奨）
- 「なぜ成功/失敗したのか」のメカニズム分析
- 数字・ROI・効果測定があれば記載

### セクション6: 🤖 AI TRANSFORMATION LENS（AI変革の視点）
- このトピックがAI変革にどう直結するか
- AI活用の具体的ユースケース2-3個
- 「AIアーキテクトとしてこの知識をどう使うか」の示唆

### セクション7: 🗺️ MENTAL MODEL（メンタルモデル）
- テキストベースの概念図（Mermaid記法 or ASCII）
- 主要概念間の関係を可視化

### セクション8: 🏋️ SYNTHESIS CHALLENGE（統合チャレンジ）
- Bloomの上位レベル（分析/評価/創造）の問い 3問
- 正答は記載しない（検索練習で使うため）
- 例: 「心理的安全性とAIエージェントチーム設計の共通原理は何か？」

### セクション9: 🧭 FURTHER EXPLORATION（探求の地図）
- 必読論文 2-3件（著者, 年, タイトル）
- 推奨書籍 1-2冊
- 関連トピックへのリンク（トピックIDで参照）

## 品質基準

| 基準 | チェック |
|------|---------|
| 正確性 | 論文の年・著者・数値が正確か |
| 面白さ | HOOKで「続きが読みたい」と思えるか |
| 実用性 | ビジネスの現場で使えるか |
| 批判性 | 限界・反論も公正に提示しているか |
| 接続性 | 他トピックとの関連が明記されているか |
| 長さ | 各セクション適切な分量か（合計2000-3000字/ターン） |
